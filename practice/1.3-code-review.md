# Code Review: Заметки

## 3 ключевых принципа Code Review

1. **Ревью — это проверка, а не переписывание.** Цель — убедиться, что код решает задачу, а не навязать свой стиль.
2. **Маленькие PR ревьюить легче.** Чем меньше изменений — тем быстрее и качественнее ревью.
3. **Комментарии должны быть конструктивными.** Не "это плохо", а "предлагаю сделать так, потому что..."

## Чеклист для ревью PR

- [ ] PR связан с Issue (есть `Closes #N`)
- [ ] Изменены только нужные файлы
- [ ] Код соответствует acceptance criteria из Issue
- [ ] Нет лишних зависимостей или файлов
- [ ] Коммит-сообщения понятные
