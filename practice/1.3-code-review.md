# Code Review Cheatsheet

## Три типа ревью

| Тип | Когда использовать |
|-----|-------------------|
| **Approve** | PR решает задачу, код корректен, замечаний нет |
| **Request Changes** | Есть блокирующие проблемы — нужно исправить перед мержем |
| **Comment** | Незначительные замечания или вопросы без блокировки мержа |

---

## Чеклист для проверки PR от агента

- [ ] PR связан с Issue (есть `Closes #N` в теле)
- [ ] Изменены только нужные файлы (не затронуты лишние)
- [ ] Код соответствует всем acceptance criteria из Issue
- [ ] Нет лишних зависимостей или сгенерированных файлов
- [ ] Коммит-сообщения понятные и соответствуют конвенции
- [ ] Ветка создана из актуального `main`
- [ ] Нет конфликтов с `main`

---

## Команды для ревью

```bash
# Просмотр PR с описанием и статусом
gh pr view 42

# Просмотр diff — все изменения в коде
gh pr diff 42

# Оставить Approve
gh pr review 42 --approve

# Запросить изменения с комментарием
gh pr review 42 --request-changes --body "Не хватает Closes #N в теле PR"

# Оставить комментарий без блокировки
gh pr review 42 --comment --body "Можно вынести в отдельную функцию"
```
