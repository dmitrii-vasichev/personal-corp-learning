# Урок 1.3 — Code Review

## Зачем нужен Code Review
- Ловит баги, которые автор не видит
- Передаёт знания команде (или тебе из контекста агента)
- Контроль качества кода агента — агент может "решить" задачу, но плохо

## Чеклист для ревью PR

- [ ] PR решает именно то, что описано в Issue
- [ ] Нет лишних файлов и изменений
- [ ] Понятные имена переменных и функций
- [ ] Нет захардкоженных секретов и ключей
- [ ] Нет дублирования кода
- [ ] PR не слишком большой (< 400 строк идеал)
- [ ] Есть тесты (если применимо)

## Команды gh для Code Review

```bash
# Посмотреть список PR
gh pr list

# Посмотреть конкретный PR (описание, статус, чеки)
gh pr view <номер>

# Посмотреть diff — что именно изменилось
gh pr diff <номер>

# Оставить комментарий к PR
gh pr comment <номер> --body "текст комментария"

# Запросить изменения (request changes)
gh pr review <номер> --request-changes --body "что исправить"

# Одобрить PR (approve)
gh pr review <номер> --approve --body "LGTM!"

# Смержить PR после approve
gh pr merge <номер> --merge
```

## Три типа результата ревью

| Результат | Когда использовать |
|---|---|
| **Approve** | Всё ок, можно мержить |
| **Request Changes** | Есть проблемы, нужны исправления |
| **Comment** | Вопрос или предложение, не блокирует |

## Инсайт
В Personal Corp ты сам ревьюер своих агентов. Агент — это "джун", который быстро пишет код, но нуждается в проверке. Code Review — твой главный инструмент контроля качества.
