# Task Manager — Learning Project

## Project
Учебный таск-менеджер: создание/назначение задач, статусы, дедлайны. Цель — отработать full-stack паттерны с Next.js App Router + Prisma.

---

## Stack
- **Frontend**: Next.js 14 (App Router), TypeScript, Tailwind CSS
- **Backend**: Next.js Route Handlers (`/app/api/`)
- **ORM**: Prisma + PostgreSQL (локально через Docker)
- **Auth**: NextAuth.js v5 (credentials provider)
- **Validation**: Zod
- **Tests**: Vitest + React Testing Library

---

## Commands

```bash
# Установка
npm install

# База данных (поднять PostgreSQL)
docker compose up -d

# Применить миграции
npx prisma migrate dev

# Сгенерировать Prisma Client
npx prisma generate

# Заполнить тестовыми данными
npx prisma db seed

# Запуск dev-сервера
npm run dev

# Запуск тестов
npm run test

# Проверка типов
npm run typecheck

# Линтер
npm run lint
```

## Architecture

```
app/
  (auth)/           # Страницы логина/регистрации (публичные)
  (dashboard)/      # Защищённые страницы приложения
    tasks/          # Список задач, детальная страница
    projects/       # Проекты
  api/
    tasks/          # Route Handlers для задач (CRUD)
    auth/           # NextAuth endpoints

components/
  ui/               # Переиспользуемые примитивы (Button, Input, Modal)
  tasks/            # Компоненты, специфичные для задач
  layout/           # Header, Sidebar, PageWrapper

lib/
  prisma.ts         # Singleton Prisma Client
  auth.ts           # Конфигурация NextAuth
  validations/      # Zod-схемы по сущностям

prisma/
  schema.prisma     # Модели БД
  seed.ts           # Тестовые данные
  migrations/       # Версионированные миграции
```

## Rules

**Компоненты**
- Используй Server Components по умолчанию; `"use client"` — только если нужен state/эффекты
- Один файл = один компонент; имя файла совпадает с именем компонента
- Props всегда типизированы через `interface`, не `type`

**API**
- Все Route Handlers валидируют входные данные через Zod перед обращением к БД
- HTTP-статусы обязательны: 200/201 success, 400 validation error, 401 unauthorized, 404 not found
- Запросы к БД — только через `lib/prisma.ts`, не создавай новый `PrismaClient` в компонентах

**Данные**
- Все мутации (create/update/delete) оборачивай в `try/catch` с логированием ошибки
- Не передавай `password` из Prisma в клиент — исключай явно через `select` или деструктуризацию

**Тесты**
- Каждый Route Handler покрыт минимум одним happy path тестом
- Тесты компонентов — только для компонентов с `"use client"` и бизнес-логикой
