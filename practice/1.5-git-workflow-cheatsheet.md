# Git Workflow Cheatsheet

## Naming conventions для веток

| Префикс | Когда использовать | Пример |
|---------|-------------------|--------|
| `feature/` | Новая функциональность | `feature/12-add-login` |
| `fix/` | Исправление бага | `fix/15-broken-redirect` |
| `chore/` | Служебные задачи | `chore/20-update-deps` |
| `docs/` | Документация | `docs/3-add-readme` |
| `practice/` | Учебные задачи | `practice/8-git-workflow` |

Правило: **префикс + номер Issue + краткое описание**

## Полный цикл: Issue → Merge

1. **Создать Issue** — описать задачу (`gh issue create`)
2. **Создать ветку** от main с номером Issue (`git checkout -b feature/N-name`)
3. **Сделать изменения** и закоммитить (`git add` → `git commit`)
4. **Запушить ветку** на GitHub (`git push -u origin branch-name`)
5. **Создать PR** со ссылкой на Issue (`gh pr create`, в теле `Closes #N`)
6. **Code Review** — проверить diff (`gh pr diff`)
7. **Смержить PR** (`gh pr merge --merge`)
8. **Вернуться на main** и подтянуть (`git checkout main && git pull`)

## Ключевые команды

### Git
- `git checkout -b name` — создать ветку и переключиться
- `git branch` — показать все ветки
- `git add file` — добавить файл в staging
- `git commit -m "msg"` — сделать коммит
- `git push -u origin branch` — запушить ветку (первый раз)
- `git log --oneline --graph` — компактная история коммитов
- `git checkout main && git pull` — вернуться на main и обновить

### GitHub CLI (gh)
- `gh issue create --title "..." --body "..."` — создать Issue
- `gh pr create --title "..." --body "..."` — создать PR
- `gh pr view` — посмотреть PR
- `gh pr diff` — посмотреть изменения в PR
- `gh pr review --approve` — одобрить PR (чужой)
- `gh pr merge --merge` — смержить PR

## Магические слова для автозакрытия Issues

В теле PR любое из этих слов + `#N` закроет Issue автоматически при мерже:
- `Closes #N`
- `Fixes #N`
- `Resolves #N`

## Чеклист Code Review

- [ ] Код делает то, что описано в Issue?
- [ ] Нет лишних изменений (мусорных файлов, отладочного кода)?
- [ ] Понятные имена переменных и функций?
- [ ] Нет дублирования кода?
- [ ] Нет проблем с безопасностью?
- [ ] Тесты проходят (если есть)?
