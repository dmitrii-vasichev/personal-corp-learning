# Module 2: GitHub Issues — Cheatsheet

## Анатомия хорошего Issue

Хороший Issue содержит:
- **Title** — краткое описание (императив: "Add", "Fix", "Update")
- **Description** — что нужно сделать и зачем
- **Acceptance Criteria** — список критериев готовности (чекбоксы)
- **Технические заметки** — детали реализации

**Пример:**

```
title: feat: Add user authentication

## Описание
Добавить авторизацию через GitHub OAuth.

## Зачем
Пользователи должны входить через свой GitHub аккаунт.

## Acceptance Criteria
- [ ] OAuth flow реализован
- [ ] Токен сохраняется в сессии
- [ ] Logout работает корректно
```

## Issue Templates

Шаблоны хранятся в `.github/ISSUE_TEMPLATE/`. Каждый файл начинается с **front matter**:

```yaml
---
name: Bug report
about: Report a reproducible bug
title: "fix: "
labels: bug
assignees: ""
---

## Описание бага
...
```

Несколько шаблонов → каждый в отдельном файле → GitHub показывает выбор при создании Issue.

## Closing Keywords

При упоминании в коммите или PR автоматически закрывают связанный Issue:

| Keyword        | Пример                        |
|----------------|-------------------------------|
| `closes`       | `Closes #19`                  |
| `fixes`        | `Fixes #42`                   |
| `resolves`     | `Resolves #7`                 |
| `close`        | `Close #100`                  |
| `fix`          | `Fix #3`                      |
| `resolve`      | `Resolve #55`                 |

Работают в:
- Теле PR (закрывает при мерже в main)
- Сообщении коммита (закрывает сразу при пуше в default branch)

## Labels / Milestones

### Labels — gh CLI

```bash
# Создать label
gh label create bug --color d73a4a --description "Something isn't working"

# Добавить label к Issue
gh issue edit 19 --add-label documentation

# Убрать label
gh issue edit 19 --remove-label "in progress"

# Список всех labels
gh label list
```

### Milestones — gh CLI

```bash
# Создать milestone
gh api repos/{owner}/{repo}/milestones \
  --method POST \
  -f title="Module 2: Issues" \
  -f due_on="2026-03-31T00:00:00Z"

# Назначить milestone к Issue
gh issue edit 19 --milestone "Module 2: Issues"

# Список milestones
gh api repos/{owner}/{repo}/milestones
```
