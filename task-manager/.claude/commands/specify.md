Создай PRD (Product Requirements Document) для следующей фичи:

$ARGUMENTS

## Инструкции

1. Прочитай CLAUDE.md проекта, чтобы понять контекст и стек
2. Создай файл `docs/prd-<short-name>.md` с PRD по следующей структуре:

### Структура PRD:
- **Проблема** — какую проблему решаем (2-3 предложения)
- **Цель** — что хотим получить (конкретный результат)
- **Пользовательские сценарии** — как пользователь будет взаимодействовать (2-3 сценария)
- **Функциональные требования** — что система должна делать (пронумерованный список)
- **Вне scope** — что явно НЕ входит в эту фичу
- **Технический дизайн** — как реализовать в рамках текущего стека
- **Критерии готовности** — как проверить что фича завершена (чеклист)

3. Каждый критерий готовности должен быть проверяемым: да/нет, без субъективных оценок
4. Секция "Вне scope" обязательна — она ограничивает работу агента
5. Покажи готовый PRD и спроси, нужны ли правки
