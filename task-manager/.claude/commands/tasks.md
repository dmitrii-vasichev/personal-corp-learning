Создай GitHub Issues на основе плана реализации.

## Вход
Прочитай план реализации: $ARGUMENTS

## Инструкции

1. Прочитай указанный план реализации целиком
2. Для каждого шага из плана создай отдельный GitHub Issue через `gh issue create`
3. Каждый Issue должен содержать:

### Структура Issue:
- **Title:** краткое описание задачи
- **Body:**
  - Описание: что нужно сделать и зачем (2-3 предложения)
  - Acceptance Criteria: чеклист проверяемых критериев (`- [ ]`)
  - Technical Notes: какие файлы затрагивает, на что обратить внимание
  - Ссылка на PRD и план

4. Добавь подходящие labels (feature, bug, chore, documentation)
5. Если есть зависимости между задачами — укажи в описании "Depends on #N"
6. После создания всех Issues выведи сводку: номера, названия, зависимости
