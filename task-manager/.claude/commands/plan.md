Создай план реализации на основе PRD.

## Вход
Прочитай PRD-документ: $ARGUMENTS

## Инструкции

1. Прочитай CLAUDE.md проекта, чтобы понять стек и архитектуру
2. Прочитай указанный PRD-документ целиком
3. Создай файл `docs/plan-<feature-name>.md` с планом реализации:

### Структура плана:
- **Обзор** — что реализуем (1-2 предложения, ссылка на PRD)
- **Шаги реализации** — пронумерованный список задач, каждая содержит:
  - Название задачи
  - Что конкретно нужно сделать
  - Какие файлы затрагивает
  - Зависимости (от каких шагов зависит)
- **Порядок выполнения** — в каком порядке выполнять шаги (с учётом зависимостей)
- **Риски** — что может пойти не так (1-3 пункта)

4. Каждый шаг должен быть достаточно атомарным для одного Issue (один агент, один PR)
5. Если шаг слишком большой (затрагивает >5 файлов) — разбей на подшаги
6. Покажи готовый план и спроси, нужны ли правки
