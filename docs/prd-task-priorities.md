# PRD: Приоритеты задач

## 1. Проблема

Сейчас все задачи в Task Manager выглядят одинаково — нет способа понять, какая задача важнее. Пользователь вынужден держать приоритеты в голове или во внешних инструментах.

## 2. Цель

Пользователь может назначить задаче приоритет (high / medium / low) и сразу видеть, за что браться в первую очередь. Список задач можно фильтровать по приоритету.

## 3. Пользовательские сценарии

- Пользователь создаёт задачу → она получает приоритет "medium" по умолчанию
- Пользователь открывает задачу → меняет приоритет на "high" или "low"
- Пользователь смотрит список задач → видит визуальный индикатор приоритета рядом с каждой задачей
- Пользователь фильтрует список → видит только задачи с выбранным приоритетом

## 4. Требования

### Функциональные

- [ ] Три приоритета: high, medium, low
- [ ] Приоритет по умолчанию при создании: medium
- [ ] UI для смены приоритета (кнопка или dropdown)
- [ ] Фильтр по приоритету в списке задач
- [ ] Визуальное отличие задач по приоритету (цвет/иконка: 🔴 high, 🟡 medium, 🟢 low)

### Нефункциональные

- [ ] Смена приоритета без перезагрузки страницы

## 5. Scope

### В scope

- Три фиксированных приоритета
- Фильтрация по одному приоритету

### Вне scope

- Числовые приоритеты и произвольные уровни (будет в v2)
- Сортировка списка по приоритету (будет в v2)
- Множественная фильтрация (по приоритету + по статусу)
- Уведомления и эскалация при просрочке high-задач

## 6. Дизайн

Приоритет хранится в поле `priority` в объекте Task:

```ts
type Task = {
  id: string
  title: string
  status: "todo" | "in_progress" | "done"
  priority: "high" | "medium" | "low"  // новое поле
}
```

## 7. Метрики успеха

- Все 4 сценария работают
- Приоритет сохраняется после перезагрузки страницы
- Визуальные индикаторы отображаются корректно для всех трёх значений
